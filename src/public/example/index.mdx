## Introduce

In this section we create new package `note` of company `company`. The Notes app will allow users to post notes and sharing with others users.

- Allows user sharing note, including attachments, privacy.
- Configure settings, includes title, description, tags, category, etc.
- Configure permissions, admin assigns permission based on user roles.
- View notes of user's friends on their activity streams.
- Getting notifications when others comment, like user notes.

## Add New Model

### Backend

Access `/admincp`, Visit `Apps` -> `Installed`, Press `Code Generator` -> `Model`. Fill your package name and previous schemas, etc "notes".

MetaFox generates somes classes based on what features you have choose on the previous form.

** Has Repository? **

Generate files for repository pattern associate with the model. the pattern based on [l5-repository](https://github.com/andersao/l5-repository)

** Has Model Factory? **

Generate files for [Model Factory](https://laravel.com/docs/9.x/database-testing#defining-model-factories).

** Has Authorization ? **

Add permissions pivot for the model based on [laravel-permission](https://spatie.be/docs/laravel-permission/v5/introduction)

** Has Text Data? **

Separate text to the seconds schemas, it's helpful to reduce size of main schema.

** Has Category Data **

Create a pivot model associate with main schema to storage relation between main schema and a category schema.

** Has Tags Data **

Create a pivot schema associate with main schema to storage relation between main schema and a tags data.

** Has Activity Feed? **

Create a pivot schema to publish each note item to activity stream.

** Has Model Observer? **

Create [Observer](/backend/eloquent#observers) to listen modified of main schema.

> Check source base of generated code, auto generated code create skeletons to save your time.

```tsx
/**
 * @type: formElement
 * name: form.element.textarea
 */
import MuiTextField from "@mui/material/TextField";
import { useField } from "formik";
import { camelCase } from "lodash";
import { createElement } from "react";
import { FormFieldProps } from "../types";

const TextAreaField = ({
  config,
  disabled: forceDisabled,
  name,
  formik,
}: FormFieldProps) => {
  const [field, meta] = useField(name ?? "TextField");
  const {
    label,
    disabled,
    labelProps,
    placeholder,
    variant,
    margin = "normal",
    fullWidth,
    type = "text",
    rows = 5,
    description,
    autoFocus,
    required,
    maxLength,
  } = config;

  // fix: A component is changing an uncontrolled input
  if (!field.value) {
    field.value = config.defaultValue ?? "";
  }

  const haveError = Boolean(meta.error && (meta.touched || formik.submitCount));

  return createElement(MuiTextField, {
    ...field,
    required,
    multiline: true,
    disabled: disabled || forceDisabled || formik.isSubmitting,
    variant,
    label,
    "data-testid": camelCase(`field ${name}`),
    autoFocus,
    inputProps: { "data-testid": camelCase(`input ${name}`), maxLength },
    rows,
    InputLabelProps: labelProps,
    placeholder,
    margin,
    error: haveError ? meta.error : false,
    fullWidth,
    type,
    helperText: haveError ? meta.error : description,
  });
};

export default TextAreaField;
```

** @type: formElement **

This annotation mark the file is define a form field component, build tool collects it to bundle all files into a chunks.

** name: form.element.textarea **

When the form is return from api, form builder detect use this component to render element has "component": "textarea" key/value pair.

## Global Search

Global search system is centralize searchable content system, you must extract which content is searchable by implement `MetaFox\Platform\Contracts\HasGlobalSearch` to your main modal. In this example, extends `Node` model by HasGlobalSearch

```php
<?php

namespace MetaFox\Note\Models;

// declares php "uses" directive.

class Note extends Model implements HasGlobalSearch // , and other interfaces
{

    // others property and method
    public function toSearchable(): ?array
    {
        // A draft blog is not allowed to be searched
        return [
            'title' => $this->title,
            'text'  => 'content of your text',
            'category' => '',
            // others data.
        ];
    }
}
```

Search system has event listener listens on modification of Note data and update its data in queue worker.

## Activity Feed

To support activity feed system, implement Note model with `MetaFox\Platform\Contracts\ActivityFeedSource` and `MetaFox\Platform\Contracts\HasResourceStream`.

```php
<?php

namespace MetaFox\Note\Models;
use MetaFox\Platform\Support\FeedAction;

// declares php "uses" directive.

class Note extends Model implements ActivityFeedSource // , and other interfaces
{
    /**
     * Define content of feed action put to activity streams
     *
     * @return FeedAction
     */
    public function toActivityFeed(): ?FeedAction
    {
        if ($this->isDraft()) {
            return null;
        }

        return new FeedAction([
            'user_id'    => $this->userId(),
            'user_type'  => $this->userType(),
            'owner_id'   => $this->ownerId(),
            'owner_type' => $this->ownerType(),
            'item_id'    => $this->entityId(),
            'item_type'  => $this->entityType(),
            'type_id'    => $this->entityType(),
            'privacy'    => $this->privacy,
        ]);
    }

    /**
     * Define morph map to privacy streams.
     */
    public function privacyStreams(): HasMany
    {
        return $this->hasMany(PrivacyStream::class, 'item_id', 'id');
    }
}
```
